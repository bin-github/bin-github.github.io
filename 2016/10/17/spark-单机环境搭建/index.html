spark 单机环境搭建

发表于 2016-10-17   |   分类于 编程

spark单机版wordcount实例运行过程
搭建linux下spark环境
java环境
下载java7或者更高的版本
在linux下解压
配置环境变量：JAVA_HOME 和 PATH 两个变量
测试是否成功：java -version 是否是和你安装的版本相同
scala环境
下载2.11.x之后的版本
接下来的步骤和安装java的相同
最后使用scala -version测试是否安装成功
hadoop环境
此处可参考这篇博文http://blog.csdn.net/zwx19921215/article/details/41819941
spark环境
下载spark2.0.1稳定版本
在linux下解压，并配置环境变量 SPARK_HOME 和 PATH
使用spark_shell命令看能否正常启动，能则安装成功
编写程序并打包
搭建开发环境：
使用idea+maven创建一个简单的java module即可
编写程序，这里使用最简单的wordCount例子
import org.apache.spark.api.java.JavaPairRDD;
import org.apache.spark.api.java.JavaRDD;
import org.apache.spark.api.java.function.FlatMapFunction;
import org.apache.spark.api.java.function.Function2;
import org.apache.spark.api.java.function.PairFunction;
import org.apache.spark.sql.SparkSession;
import scala.Tuple2;
import java.util.Arrays;
import java.util.Iterator;
import java.util.List;
import java.util.regex.Pattern;
public final class JavaWordCount {
private static final Pattern SPACE = Pattern.compile(“ “);
public static void main(String[] args) throws Exception {
if (args.length != 2) {
System.err.println(“参数输入有误，一个是输入地址，一个是输出地址”);
System.exit(1);
}
SparkSession spark = SparkSession
.builder()
.appName(“JavaWordCount”)
.getOrCreate();

    JavaRDD<String> lines = spark.read().textFile(args[0]).javaRDD();
    JavaRDD<String> words = lines.flatMap(new FlatMapFunction<String, String>() {
        @Override
        public Iterator<String> call(String s) {
            return Arrays.asList(SPACE.split(s)).iterator();
        }
    });
    JavaPairRDD<String, Integer> ones = words.mapToPair(
            new PairFunction<String, String, Integer>() {
                @Override
                public Tuple2<String, Integer> call(String s) {
                    return new Tuple2<>(s, 1);
                }
            });
    JavaPairRDD<String, Integer> counts = ones.reduceByKey(
            new Function2<Integer, Integer, Integer>() {
                @Override
                public Integer call(Integer i1, Integer i2) {
                    return i1 + i2;
                }
            });
    counts.saveAsTextFile(args[1]);
    List<Tuple2<String, Integer>> output = counts.collect();
    for (Tuple2<?,?> tuple : output) {
        System.out.println(tuple._1() + ": " + tuple._2());
    }
    spark.stop();
}

}

打包，如下图：
如下
找到你要打包的module，并指定那个是主类
如果打过包的有可能会包这个错，直接删掉改文件即可，打包完成之后也要删掉这个目录和下面的这个.mf文件，不然运行时会报错。
配置打包输出路径
选中module，并make
在对应的输出路径中找到jar包即可
在spark中运行
将打包好的jar copy到linux下的某个路径下，可在spark目录下新建一个专门存放应用的文件夹
启动spark：
切换到spark目录下，使用sbin/start-all.sh命令启动spark，如果无误，则显示如下
使用命令跑打包好的程序jar：上面的例子中稍微修改了下自带的程序，所以最后两个参数是必须的，一个是hdfs上的输入路径，一个是输出路径，这些路径可以换成普通的文件路径

bin/spark-submit –class test.JavaWordCount –master local ./apps/spark.jar hdfs://192.168.11.133:9000/data/wordCount/wordcount.txt hdfs://192.168.11.133:9000/output/count/

命令详解：
spark-submit: spark的命令
–class ：主类的文件路径，包名+类名
–master：主机的地址 + 打包文件的路径，这里使用本机local[2],中括号里的是主机的核数，可选；后面是jar包的存放路径
程序的启动参数配置，第一个是输入路径，第二个是输出路径，这个不同的工程有不同的参数设置。
参数设置具体可看官网：http://spark.apache.org/docs/latest/submitting-applications.html
如果报错：Invalid signature file digest for Manifest main attributes原因是没有删除jar包中的META-INF/.RSA META-INF/.DSA META-INF/.SF文件及文件夹：
可以直接在windows下打开jar文件直接删除
可以使用linux命令 删除：zip -d .jar META-INF/.RSA META-INF/.DSA META-INF/.SF
